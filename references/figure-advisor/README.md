# 科研数据可视化顾问

> 基于 scipilot-figure-skill 的"先思考后绘制"理念整合。
> 核心思想：**永远先思考再画**，先理解数据再选图，先想清楚"这张图要论证什么"。

---

## 概述

科研工作者最大的画图痛点往往不是"不会用 matplotlib"，而是"手上一堆数据，不知道该用什么图把结论讲清楚"。本模块的**首要能力是【思考与判断】**，其次才是【绘制】。

具体地——**永远先思考再画**：
1. 先理解数据再选图——拿到数据先做 EDA，用事实驱动图型选择
2. 先想清楚"这张图要论证什么"——同样数据，不同论点 = 不同图
3. 主动拦截科研画图的经典错误，而不是顺从
4. 维度太多就建议拆图，不硬塞

---

## 核心工作流（8 步）

```
0. 理解任务   ── 这张图要论证什么？数据在哪？
   ↓
1. 剖析数据   ── 列类型/样本量/分布/异常/相关
   ↓
2. 选图       ── 按数据形态+论证目标决策
   ↓ (主动拦截 → viz-pitfalls.md)
3. 查规范     ── 栏宽/字号/DPI/字体
   ↓
4. 配环境     ── 期刊预设+CJK字体
   ↓
5. 绘制       ── 9 类图配方
   ↓
6. 自检闭环   ── 程序自检 + AI 读图（缺字/裁切/重叠/对齐→回改）
   ↓
7. 导出       ── 多格式+按最终尺寸+灰度预览
```

详见 `figure-workflow.md`。

## 五条硬性原则

1. **按最终尺寸出图，不二次缩放** — `figsize` 直接定论文里实际尺寸
2. **矢量优先** — 数据图走 PDF / SVG / EPS；照片才用 TIFF / PNG；**绝不 JPEG**
3. **配色对色盲友好** — 默认色盲安全调色板 + 冗余编码 + 灰度预览
4. **字号在最终尺寸下可读** — 7-9 pt，最小 6 pt
5. **误差必有交代** — 图注写清 SD/SEM/95%CI + n + 检验方法

## 主动拦截（精选）

| 错误 | 后果 | 替代方案 |
|---|---|---|
| n<10/组 还想画均值柱 | 掩盖分布、掩盖 n | 箱线 + stripplot |
| 双 Y 轴显示无关变量 | 视觉上的相关是捏造的 | 拆成上下子图 |
| 用饼图展示占比 | 人眼判角度差长度 3 倍 | 横向柱状 |
| rainbow / jet 色图 | 感知不均匀、造假峰 | viridis / RdBu_r |
| 一图塞 5 个论点 | 没论点 | 拆图，一张图一个结论 |

完整 15 条详见 `viz-pitfalls.md`。

## 何时使用

- 用户给了一个 CSV / Excel / DataFrame 说"帮我画一下"或"用什么图好"
- 用户在写论文要插数据图
- 用户已有草图但说"达不到投稿要求"
- 用户提到 Nature / Science / IEEE 等具体期刊
- 用户问"中文论文 matplotlib 出方框怎么办"
- 用户提到误差棒、显著性、色盲、矢量导出、多面板

**即使用户只是给一批数据问"这个怎么画"或"用什么图好"，也应使用本模块——首要能力是"判断该用什么图"，其次才是绘制。**

## 数据形态 → 推荐图型 速查

| 数据形态 | 首选 | 备选 | 不该用 |
|---|---|---|---|
| 单个连续看分布 | KDE / 直方图 | 箱线 / 小提琴 | 饼图 / 折线 |
| 单个分类看占比 | 横向柱状（按值排序） | 堆叠柱状 | **饼图** |
| 1 分类 vs 1 连续，n≥10/组 | 箱线 + stripplot | 小提琴 / 带误差柱 | 仅画均值柱 |
| 1 分类 vs 1 连续，n<10/组 | stripplot / dot plot | 蜂巢散点 | 箱线 / 均值柱 |
| 2 连续 | 散点 + 回归线 | 2D KDE / hexbin | 折线（除非有序） |
| 时间 vs 连续 | 折线 + 误差带 | 阶梯图 / 散点 | 柱状 |
| 多连续 | 相关性热力图 | pairplot | 平行坐标 |
| 矩阵数据 | 热力图（感知均匀色图）| 表格 | 3D 表面 |

详见 `chart-selection.md`。
