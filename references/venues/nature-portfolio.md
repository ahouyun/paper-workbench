# Nature Portfolio 期刊规范

## 期刊家族

### Nature 主刊

**定位**：发表具有广泛科学兴趣的突破性研究

**风格要求**：
- 创新性优先：必须有明确的 novelty claim
- 广泛兴趣：研究问题需面向广泛读者，不限于专业领域
- 简洁表达：正文通常 3000-5000 词
- 证据链完整：每个声明需有实验支持

**结构**：
- Title: ≤90 字符（含空格）
- Abstract: ≤150 词，无引用
- Main text: 3000-5000 词
- Methods: 可在线补充
- References: ≤50 篇（正文）

### Nature 子刊

**定位**：专业领域的高质量研究

**风格要求**：
- 专业深度：方法论需更详细
- 严谨性：统计报告需完整
- 创新性：仍需明确 novelty

**主要子刊**：
- Nature Medicine: 临床和转化研究
- Nature Biotechnology: 方法和应用
- Nature Communications: 开放获取，范围更广
- Nature Machine Intelligence: AI/ML 研究
- Nature Methods: 方法学创新

### Science

**定位**：美国科学促进会旗舰期刊

**风格要求**：
- 突破性发现
- 证据链完整
- 简洁有力

**结构**：
- Title: 简洁
- Abstract: ≤250 词
- Main text: ≤4500 词
- References: ≤40 篇

### Cell

**定位**：生命科学机制研究

**风格要求**：
- 机制深入：需多角度验证
- 证据链：功能实验 + 机制实验 + 体内验证
- 图表丰富：通常 6-8 个主图

## 通用写作规范

### 语言要求

1. **简洁性**：每句 ≤30 词
2. **清晰性**：避免过度技术化术语
3. **英式英语**：signalling, colour, analyse, programme, modelling, behaviour
4. **主动语态**：优先使用 "We show" 而非 "It was shown"

### 结构要求

1. **倒金字塔**：从广泛背景到具体发现
2. **故事弧**：Context → Gap → Discovery → Implication
3. **Figure 1**：核心发现的可视化
4. **逻辑流畅**：段落间有明确过渡

### 引用要求

1. **正文引用**：作者+年份格式 (Smith et al., 2024)
2. **参考文献列表**：按出现顺序编号
3. **DOI 必需**：每条引用必须有 DOI
4. **数量限制**：正文 ≤50 篇引用

## 数据和方法

### 数据可用性声明

**必需**：所有论文必须包含 Data Availability Statement

**格式**：
```
Data availability
The data that support the findings of this study are available from [repository] 
at [URL/DOI]. The code is available at [GitHub URL].
```

### 方法要求

1. **可重复性**：足够细节供他人重复
2. **统计报告**：样本量、p 值、置信区间、效应量
3. **伦理声明**：涉及人类或动物研究需伦理批准
4. **利益冲突**：必须声明

## 图表规范

### 图表数量

- Nature 主刊：通常 4-6 个主图
- 子刊：根据需要，通常 4-8 个
- 补充材料：不限

### 图表格式

- **格式**：TIFF, EPS, PDF（矢量图优先）
- **分辨率**：≥300 dpi（彩图），≥600 dpi（线条图）
- **字体**：Arial, Helvetica, sans-serif
- **字号**：≥5pt（最终尺寸）

### 图表设计

1. **信息层次**：三级结构（主信息 → 次信息 → 细节）
2. **颜色**：色盲友好调色板
3. **简洁性**：避免装饰性元素
4. **自包含**：图注应能独立理解

## 审稿流程

### 审稿标准

1. **创新性**：是否提供新知识
2. **严谨性**：方法和证据是否可靠
3. **广泛兴趣**：是否面向广泛读者
4. **清晰性**：写作是否清晰

### 审稿人选择

- 2-3 个审稿人
- 领域专家 + 跨学科视角
- 避免利益冲突

### 决策类型

- **Accept**: 直接接受
- **Minor Revision**: 小修后接受
- **Major Revision**: 大修后重审
- **Reject**: 拒稿
