# DOCX 交付

将论文草稿转换为 `.docx`，支持四种交付路径。

## 核心原则

能复制/编辑时保留原 DOCX 格式；不能时生成结构良好的 Word 文档。

## 四种交付路径

### M1：默认样式生成

**输入**：Markdown 草稿
**输出**：新 `.docx`

使用内置默认格式：
- 正文：宋体 12pt
- 标题：黑体
- 行距：1.5 倍
- 首行缩进：0.74cm

适用于无模板、无样文的场景。

### M2：仿样文样式

**输入**：Markdown 草稿 + 样文分析 JSON
**输出**：新 `.docx`

从样文提取高置信样式（sample_count ≥ 3），深度合并到默认配置：
- 字体、字号
- 缩进、行距
- 标题层级样式

使用 `scripts/format_checker.py` 分析样文生成分析 JSON。

### M3：学校模板填充（推荐）

**输入**：Markdown 草稿 + 学校模板 `.docx` + 规格 YAML
**输出**：模板副本填充

**核心逻辑**：复制模板 OOXML 基础，只替换占位文本 + 插入章节内容，不修改样式。

适用于有严格格式要求的学校。模板源文件永远不被修改。

**填充报告**：输出被替换的占位符、未匹配字段、章节配对方式、需人工审查项。

### M4：现有草稿编辑

**输入**：草稿 `.docx` + 编辑指令
**输出**：同一 `.docx`

仅修改段落文本，保留 pPr/rPr/sectPr/comments。

编辑指令格式：
- `--replace "锚点文本" "新文本"`
- `--insert-after "锚点段落" "新段落"`
- `--delete "要删除的段落锚点"`

锚点必须唯一匹配（0 或多个匹配时报错）。

---

## 支持范围

默认 DOCX 处理：
- 标题（1-3 级）
- 摘要、关键词
- 正文段落
- 图片（通过 image map 插入）
- 表格（Markdown 表格转 Word 表格）
- 公式（`latex_text` 模式或 `formula_image` 模式）
- 参考文献
- 致谢
- 附录
- 代码块
- 缺失资产占位内容

## 公式处理

| 模式 | 说明 |
|------|------|
| `latex_text`（默认） | 保留 LaTeX 文本 |
| `formula_image` | 渲染为图片插入 |

## 资产和清理规则

- 截图、图表和公式图片通过 image map 插入
- Markdown 表格应转换为 Word 表格；不可解析时内容必须保留或明确标记——**不能静默丢失**
- Mermaid/PlantUML/draw.io 源码放入附录 DOCX
- 导出前所有 Markdown 语法必须清理

## 预导出检查

- [ ] 无残留 `[image_N]` 占位符
- [ ] 所有引用图片文件存在且非空
- [ ] 参考文献列表完整
- [ ] 图表编号连续无跳号
- [ ] 章节顺序正确

## 导出失败处理

- 缺失图片：使用红色占位文字（"图片缺失：文件名"），不中断导出
- 格式解析失败：保留原始 Markdown 内容，标记格式异常
