# AIGC 治理策略

合并中英文两种模式的 AIGC 治理策略。

## 核心原则

**不追求检测工具分数，追求语义准确 + 学术严谨。**

禁止：简单同义替换、删词式压缩、追逐检测分数。

---

## 中文模式（毕设论文）

### 治理流水线

场景化重写 → 结构重组 → 细节注入 → 自然承接与轻冗余 → 高密度句拆句解释 → 语言去模板化 → 自检清单

### 章节策略矩阵

| 章节 | 策略 | 说明 |
|------|------|------|
| 摘要 | 保守 | 保持精炼，少量自然化 |
| 绪论 | 中等 | 注入研究动机和背景细节 |
| 技术章节 | 保守 | 保持技术准确性 |
| 设计章节 | 中等 | 注入设计决策理由 |
| 实现章节 | 积极 | 注入实现细节和踩坑经验 |
| 测试章节 | 中等 | 注入测试策略理由 |
| 结论 | 保守 | 保持概括性 |
| 致谢 | 积极 | 可注入个人情感 |

### 禁止行为

- 禁止简单同义替换作为主要手段
- 禁止删词式压缩
- 禁止引入口语化标记、刻意错别字、网络用语
- 禁止伪造引用或数据

### 输出格式

每次 AIGC 治理必须输出：
1. **处理前计划**：分析问题和改写策略
2. **改写文本**：实际改写结果
3. **自检清单**：逐项检查通过情况

自检项出现"未通过"时继续局部修正，不得交付为最终版。

### 成语使用

- 适度使用（每段 2-3 个），增加自然感
- 不得在技术细节附近使用
- 不得过度堆砌

## 英文模式（学术论文）

### 29 种 AI 写作模式检测

#### 内容模式

| 模式 | 检测信号 | 修复方式 |
|------|----------|----------|
| 重要性膨胀 | "pivotal moment", "crucial role" | 直接陈述事实和数据 |
| 名人堆砌 | 列举会议/期刊名而不引用具体工作 | 引用具体论文 |
| 表面分析 | "showcasing", "highlighting" | 替换为具体声明 |
| 推销语言 | "groundbreaking", "novel framework" | 使用具体描述词 |
| 模糊归因 | "It is widely believed" | 引用具体论文 |
| 公式化挑战 | 泛泛而谈的"challenges" | 用证据具体陈述 |

#### 语言模式

| 模式 | 检测信号 | 修复方式 |
|------|----------|----------|
| 高频 AI 词汇 | "crucial", "delve", "pivotal", "tapestry", "testament", "underscore" | 替换为精确词汇 |
| 系动词回避 | "serves as" 替代 "is" | 直接使用 "is" |
| 否定并列 | "Not only X, but also Y" | 使用简单陈述 |
| 强制三连 | 连续三个并列项 | 减少到两个或用其他结构 |
| 同义词循环 | model/framework/approach/method 交替使用同一概念 | 统一术语 |
| 被动语态过度 | 全文被动 | 适当使用主动语态 |

#### 风格模式

| 模式 | 检测信号 | 修复方式 |
|------|----------|----------|
| 破折号过度 | 频繁使用 em dash | 使用逗号或括号 |
| 粗体过度 | 大量粗体标记 | 仅在必要时使用 |
| 行内标题列表 | 段落内用粗体做小标题 | 改为正常段落 |
| 标题大小写 | Title Case in headings | 使用 Sentence case |

## 与段落清晰度的协同

AIGC 治理在段落清晰度修复之后进行：

1. 先修复结构和逻辑（段落清晰度检查）
2. 再修复 AI 写作痕迹（AIGC 治理）
3. 最后做对抗性审稿（paper-review）

## 集群检测哲学

**关键洞察：** 单个 AI 模式实例无意义；多个模式同时出现才是问题。

**检测规则：**
- **段落级：** 3+ 个不同 AI 模式出现在同一段落 = 高风险区
- **章节级：** 5+ 个不同模式出现在同一章节 = 需要重写该章节
- **严重性聚合：** 单个低严重性 + 集群 = 中严重性

**示例集群：**
> "It is worth noting that our novel approach leverages cutting-edge techniques to achieve groundbreaking results. Furthermore, extensive experiments demonstrate the effectiveness of our method."

包含："It is worth noting"（填充词）+ "novel"（过度使用）+ "leverages"（AI词汇）+ "groundbreaking"（重要性膨胀）+ "Furthermore"（填充词）+ "extensive experiments demonstrate"（模板）= **6 个模式在 2 句话中 = 高风险集群**

## 误报意识

**不要在以下情况下标记这些模式：**

| 模式 | 可接受场景 |
|------|-----------|
| "novel" | 在摘要/引言中出现一次，且有具体贡献 |
| "state-of-the-art" | 紧跟基准数字 |
| "end-to-end" | ML 系统论文中的技术术语 |
| 被动语态 | 方法部分描述实验过程 |
| "However" | 在摘要中出现一次；仅在 2+ 次时标记 |
| "comprehensive" | 列出实际范围时（如"across 5 benchmarks"） |
| Hedging 词汇 | 讨论局限性或未来工作时 |

## 按章节免检列表

**摘要：** "We propose..."、"novel"（一次）、"However"（一次）
**引言：** "In this paper, we present..."、讨论先前工作时的 hedging
**方法：** 被动语态、技术术语、"We use/adopt/employ"
**结果：** 有统计支持的"significant"、直接比较
**讨论：** Hedging（"suggests"、"may indicate"）、局限性确认
