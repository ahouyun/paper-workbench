# 动机主线写作法 (Motivation-Thread Writing)

> 来源：PaperSpine skill。好论文不是一组改进后的section，而是一个完整的问题-解决弧线。

---

## 一、核心理念

**好论文 = 一条完整的动机主线**

```
Field problem → Specific gap → Design response → Evidence → Interpretation → Limitation/Future
```

每个section都必须服务于这条主线，而不是独立存在。

---

## 二、动机主线模型

### 2.1 七锚点测试

一篇好论文必须有7个关键锚点，构成连贯的问题-解决弧线：

| 锚点 | 位置 | 作用 |
|------|------|------|
| 1. Abstract motivation | Abstract | 一句话说明为什么做这个研究 |
| 2. Introduction opening | Introduction第1段 | 建立领域重要性 |
| 3. Gap sentence | Introduction第3-4段 | 指出现有方法的具体不足 |
| 4. Contribution sentence | Introduction最后 | 列出核心贡献 |
| 5. Methods rationale | Method开头 | 解释为什么这样设计 |
| 6. First Results headline | Results开头 | 展示最重要的发现 |
| 7. Discussion closing | Discussion最后 | 回答"So what" |

**测试方法：** 把这7个句子单独拿出来，它们应该构成一个连贯的故事。

### 2.2 动机表层映射

动机必须在以下位置显式出现：

| 位置 | 如何体现动机 |
|------|-------------|
| 标题 | 核心创新点 |
| Abstract | 问题→方案→结果 |
| Introduction topic sentence | 每段第一句服务于动机 |
| Transition | 段落间过渡连接动机 |
| Methods模块命名 | 模块名反映设计意图 |
| Results headline | 结果服务于动机验证 |
| Discussion closure | 回答动机是否被满足 |

## 三、写作理由矩阵 (Writing Rationale Matrix)

### 3.1 什么是写作理由矩阵

**写作理由矩阵是写作执行计划，不是事后总结。** 每个段落都必须解释：
- 它在全文中的功能
- 它与动机的关联
- 它参考了哪些SOTA模式
- 它的目标期刊规范
- 它的证据锚点

### 3.2 矩阵模板

| Row ID | Manuscript Unit | Current/Planned Function | Motivation Link | Reference/SOTA Pattern | Target Venue Norm | Evidence Anchor | Planned Change | Final Check |
|--------|----------------|-------------------------|----------------|----------------------|-------------------|----------------|----------------|-------------|
| 0 | 全文框架 | 主线论证 | 核心动机 | - | - | - | - | - |
| 1 | Abstract | 背景+问题+方案+结果 | 动机概述 | - | 150-200词 | - | - | - |
| 2 | Intro-§1 | 领域重要性 | 动机来源 | - | - | 引用1-3 | - | - |
| 3 | Intro-§2 | 技术挑战 | 动机具体化 | - | - | - | - | - |
| ... | ... | ... | ... | ... | ... | ... | ... | - |

### 3.3 矩阵质量标准

**浅矩阵是失败的：**
- ❌ 大多数行只说"improve clarity"或"polish wording"
- ❌ 没有解释段落与动机的关联
- ❌ 没有引用SOTA模式

**深矩阵是成功的：**
- ✅ 每行解释多个维度的具体锚点
- ✅ 每个段落都服务于动机主线
- ✅ 引用了SOTA论文的写作模式

## 四、深度模仿协议 (Deep Imitation Protocol)

### 4.1 六层学习

| 层次 | 学习内容 | 示例 |
|------|---------|------|
| 1. Argument architecture | 论证结构 | "先扬后抑"、"Trade-off显式化" |
| 2. Section rhythm | 段落节奏 | 长短句交替、信息密度变化 |
| 3. Claim calibration | 声明校准 | 声明强度与证据匹配 |
| 4. Evidence placement | 证据放置 | 何时放数字、何时放引用 |
| 5. Sentence architecture | 句子结构 | 主动/被动、简单/复杂 |
| 6. Reader contract | 读者契约 | 承诺→兑现的节奏 |

### 4.2 闭卷重写

**核心规则：不编辑原文段落，而是从蓝图和笔记重写。**

**流程：**
1. 提取事实/数字/引用到笔记
2. 隐藏原文
3. 从蓝图和笔记重写
4. 仅在完整性检查时回看原文

**最低重写标准：**
- 近似段落比例 < 35%
- 主导操作不是ADD
- KEEP行 < 25%

### 4.3 七句传承测试

这7个句子必须构成连贯的故事：
1. Abstract final sentence → Introduction opening
2. Introduction opening → Gap sentence
3. Gap sentence → Contribution sentence
4. Contribution sentence → Methods rationale
5. Methods rationale → First Results headline
6. First Results headline → Discussion closing

## 五、深度阅读协议 (Deep Reading Protocol)

### 5.1 三层阅读法

**Pass 1: Paper Skeleton + Context Map**
- 5Cs分析：Context, Challenge, Contribution, Confidence, Call-to-action
- Section Tree：章节结构树
- Figure/Table Inventory：图表清单

**Pass 2: 8个结构化提取模板**
- A: Terminology Glossary（术语表）
- B: Transition Phrase Bank（过渡词库）
- C: Sentence Pattern Bank（句式库）
- D: Argument Pattern Deconstruction（论证模式解构）
- E: Abstract Rhetorical Move Map（摘要修辞动作图）
- F: Figure/Table Design Patterns（图表设计模式）
- G: Section-Level Information Conventions（章节信息规范）
- H: Rhetorical Move Analysis per Section（基于Swales CARS模型）

**Pass 3: FAT Analysis**
- Facts（事实）
- Appreciation（评价）
- Tie（关联）

### 5.2 Cross-Paper Convention Synthesis

跨论文比较8个维度，提取NON-NEGOTIABLE RULES：
1. 标题命名规范
2. Abstract结构
3. Introduction逻辑
4. Method描述方式
5. 实验设计模式
6. 结果呈现方式
7. 讨论深度
8. 结论收尾

## 六、逻辑转移审计 (Logic Transfer Audit)

### 6.1 四个核心问题

1. **原文在做什么？** — 理解原文的论证逻辑
2. **目标场景教了我们什么逻辑？** — 从SOTA论文学习
3. **修订后用了什么逻辑？** — 检查修订是否合理
4. **哪些具体段落因学习而改变？** — 追踪变更来源

### 6.2 七锚点测试

Abstract motivation → Introduction opening → Gap sentence → Contribution sentence → Methods rationale → First Results headline → Discussion closing

**这7个句子必须构成连贯的故事。**

### 6.3 缺陷标签

| 标签 | 含义 |
|------|------|
| background-stack | 堆砌背景，没有聚焦 |
| gap-vague | 空白描述模糊 |
| method-recipe | 方法描述像食谱 |
| metric-dump | 数字堆砌无分析 |
| claim-leap | 声明跳跃，缺乏证据 |
| discussion-repeat | 讨论重复结果 |
| latex-driven | 被LaTeX格式驱动 |

## 七、降AI检测率 (Humanize)

### 7.1 5个AIGC检测维度

| 维度 | AI特征 | 人类特征 |
|------|--------|---------|
| D1: 句长分布 | 单峰15-25字 | 多峰分布 |
| D2: 段落结构相似度 | 0.7-0.9 | 0.2-0.5 |
| D3: 信息密度 | 65-75%稳定 | 40-85%波动 |
| D4: 连接词分布 | 8-15个/千字 | 2-6个/千字 |
| D5: 术语语境匹配 | ~100%标准语境 | 偶尔变体 |

### 7.2 三个干预层级

| 层级 | 措施 | 适用场景 |
|------|------|---------|
| light | 改连接词和句长 | 轻微模板化 |
| medium | 加段落结构变化和密度分层 | 明显模板化 |
| heavy | 引入结构不规整和术语变体 | 严重模板化 |
