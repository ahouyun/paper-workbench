# 材料收集与清单追踪

适用于论文项目的首次材料收集和清单管理。

## 硬约束

**首次回复时先收集材料，不要开始写。**

如果用户未提供模板、样文、任务书或开题报告，先要求提供（除非用户明确说没有或要求独立进行）。收到模板/样文后先分析并报告目录、字数和格式观察。

## 输入优先级

1. 学校模板
2. 任务/开题报告
3. 明确用户要求
4. 样文
5. 项目源码
6. README
7. 旧文档

## 材料清单追踪

每项材料必须在 `paper-context/workflow/material-inventory.md` 中追踪：

```markdown
| 材料 | 优先级 | 状态 | 缺失影响 |
|------|--------|------|----------|
| 学校模板 | required | 已提供 | — |
| 任务书 | required | 缺失 | 无法确认章节要求 |
| 样文 | strongly_recommended | 缺失 | 无法参考格式 |
| 项目源码 | required | 已提供 | — |
```

优先级说明：
- `required`：缺少则阻塞，无法继续
- `strongly_recommended`：缺少则降低质量但可继续
- `optional`：可选，有则更好

## 10 项必须确认

首次回复时确认以下事项（可分批确认，不必一次全部询问）：

1. **论文类型**：毕设论文/课程论文/期刊论文/学位论文
2. **选题标题**：完整标题
3. **学校模板可用性**：是否有 `.docx` 模板文件
4. **历届样文可用性**：是否有参考样文
5. **字数要求**：总字数和各章大致分配
6. **是否必须交付 `.docx`**：还是 Markdown 即可
7. **开题报告/任务书/封面字段需求**：是否需要生成
8. **系统访问信息**：如需真实截图，是否可访问系统
9. **主文件名是否必须匹配论文标题**
10. **是否需要额外 `.docx` 附录**

## 交付路径选择

确认材料后，选择交付路径：

| 路径 | 输入 | 输出 | 适用场景 |
|------|------|------|----------|
| **M1** 默认样式 | Markdown | 新 `.docx` | 无模板，用内置默认格式 |
| **M2** 仿样文 | Markdown + 样文分析 | 新 `.docx` | 有样文，提取格式 |
| **M3** 模板填充 | Markdown + 学校模板 | 模板副本填充 | **推荐**，严格格式要求 |
| **M4** 草稿编辑 | 草稿 `.docx` + 编辑指令 | 同一 `.docx` | 已有草稿需修改 |

用户选择后记录到 `paper-context/workflow/user-decisions.md`。

## 首次回复要求

首次回复时：

1. 告知用户可直接发送本地文件路径，如 `D:\论文模板.docx`
2. 列出需要的材料清单
3. 询问论文类型和选题
4. 如果用户已提供材料，先分析再报告观察结果

## 材料分析

收到材料后，不立即写作，先分析：

- 目录完整性：章节是否齐全
- 各章大致字数分配
- 格式观察：字体、字号、行距、页边距
- `.docx` 样文优先用 `scripts/format_checker.py` 或相关工具分析

## 紧缺材料处理

当材料不足时（两项以上 required 缺失、文献池为空、仅有 README 级证据）：

1. **网络搜索**：经用户同意，搜索近期领域文献，标记 `needs_check`
2. **头脑风暴**：经用户同意，写候选想法到 `content-decisions.md`
3. 用户确认的内容进入大纲和正文；拒绝的标记 `discarded`
4. 用户都拒绝则记录到 `blocker-report.md`，在可见限制内继续
