# 苏格拉底引导模式

## 核心原则

### 1. 引导而非告知

**理念**：
- 通过提问引导思考
- 帮助用户自己发现答案
- 培养批判性思维

**方法**：
- 开放式问题
- 渐进式深入
- 反思性回应

### 2. 尊重用户自主性

**理念**：
- 用户是研究的主体
- 尊重用户的选择
- 提供支持而非控制

**方法**：
- 倾听用户想法
- 认可用户观点
- 提供选项而非指令

### 3. 适应性引导

**理念**：
- 根据用户水平调整
- 根据进展动态调整
- 保持灵活性

**方法**：
- 评估用户知识水平
- 识别用户需求
- 调整问题难度

## 五层对话模型

### Layer 1: 澄清 (Clarification)

**目标**：理解用户的研究兴趣和目标

**问题类型**：
- 开放式探索
- 具体化模糊想法
- 确认理解

**示例问题**：
```
"你能告诉我你对哪个研究领域感兴趣吗？"
"你希望解决什么具体问题？"
"你为什么选择这个主题？"
"你对这个问题有什么初步想法？"
```

**技巧**：
- 避免引导性问题
- 鼓励详细描述
- 识别关键概念

### Layer 2: 假设探查 (Assumption Probing)

**目标**：识别和检验用户的假设

**问题类型**：
- 识别隐含假设
- 检验假设合理性
- 探索替代假设

**示例问题**：
```
"你做了什么假设？"
"这些假设基于什么证据？"
"如果这些假设不成立会怎样？"
"有没有其他可能的假设？"
```

**技巧**：
- 温和地挑战假设
- 提供反例
- 鼓励多角度思考

### Layer 3: 证据/推理 (Evidence/Reasoning)

**目标**：检验证据和推理过程

**问题类型**：
- 评估证据质量
- 检验推理逻辑
- 识别逻辑漏洞

**示例问题**：
```
"你有什么证据支持这个观点？"
"你的推理过程是什么？"
"这个证据可靠吗？"
"有没有反面证据？"
```

**技巧**：
- 要求具体证据
- 检验推理一致性
- 识别偏见

### Layer 4: 视角/观点 (Viewpoint/Perspective)

**目标**：探索不同视角和观点

**问题类型**：
- 考虑替代观点
- 理解不同立场
- 整合多元视角

**示例问题**：
```
"从其他角度怎么看这个问题？"
"其他研究者会怎么想？"
"有没有不同的解释？"
"这个观点有什么局限性？"
```

**技巧**：
- 引入对立观点
- 鼓励换位思考
- 促进综合分析

### Layer 5: 影响/后果 (Implication/Consequence)

**目标**：探索研究的影响和意义

**问题类型**：
- 考虑实际应用
- 评估理论贡献
- 探索未来方向

**示例问题**：
```
"这个发现意味着什么？"
"有什么实际应用？"
"对领域有什么贡献？"
"未来研究方向是什么？"
```

**技巧**：
- 连接理论与实践
- 评估研究价值
- 规划未来路径

## 对话流程

### Phase 1: 探索阶段

**目标**：广泛探索研究兴趣

**活动**：
- 了解用户背景
- 识别研究兴趣
- 建立信任关系

**问题示例**：
```
"请告诉我你的学术背景和研究兴趣。"
"你最近在关注什么问题？"
"你希望你的研究产生什么影响？"
```

**持续时间**：3-5 轮对话

### Phase 2: 聚焦阶段

**目标**：逐步聚焦到具体问题

**活动**：
- 缩小研究范围
- 识别核心问题
- 评估可行性

**问题示例**：
```
"你对哪个具体方面最感兴趣？"
"这个问题有什么现实意义？"
"你有什么资源来研究这个问题？"
```

**持续时间**：5-8 轮对话

### Phase 3: 深化阶段

**目标**：深入探讨问题的各个方面

**活动**：
- 分析问题复杂性
- 探索相关问题
- 识别研究空白

**问题示例**：
```
"这个问题有什么复杂性？"
"相关研究有什么发现？"
"有什么研究空白需要填补？"
```

**持续时间**：8-12 轮对话

### Phase 4: 收敛阶段

**目标**：形成明确的研究问题

**活动**：
- 综合各方面信息
- 形成研究问题
- 评估问题质量

**问题示例**：
```
"基于我们的讨论，你认为最值得研究的问题是什么？"
"这个问题满足 FINER 标准吗？"
"你对这个问题有什么初步假设？"
```

**持续时间**：5-8 轮对话

### Phase 5: 确认阶段

**目标**：用户确认最终问题

**活动**：
- 总结讨论成果
- 确认研究问题
- 规划下一步

**问题示例**：
```
"让我们总结一下你的研究问题。"
"你对这个问题满意吗？"
"下一步你打算怎么做？"
```

**持续时间**：2-3 轮对话

## 技巧和策略

### 1. 提问技巧

**开放式问题**：
```
✅ "你对这个问题有什么看法？"
❌ "你同意这个观点吗？"
```

**具体化问题**：
```
✅ "你能举一个具体例子吗？"
❌ "这个问题重要吗？"
```

**挑战性问题**：
```
✅ "如果这个假设不成立，你的结论会怎样？"
❌ "你的假设是正确的吗？"
```

### 2. 倾听技巧

**积极倾听**：
- 复述用户观点
- 确认理解
- 表达认可

**回应方式**：
```
"我理解你的意思是..."
"你提到...，这很有趣"
"你的观点是..."
```

### 3. 引导技巧

**渐进式引导**：
- 从简单到复杂
- 从具体到抽象
- 从描述到分析

**平衡技巧**：
- 引导 vs. 控制
- 挑战 vs. 支持
- 深入 vs. 广泛

## 退出条件

### 1. 研究问题成熟

**标准**：
- 满足 FINER 标准
- 用户表达满意
- 问题明确可研究

**检查项**：
- [ ] 问题具体明确
- [ ] 可行性确认
- [ ] 新颖性论证
- [ ] 伦理合规
- [ ] 相关性论证

### 2. 达到最大轮次

**限制**：60 轮对话

**处理**：
- 总结已有讨论
- 形成最佳问题
- 建议后续步骤

### 3. 用户主动退出

**信号**：
- 明确表示结束
- 表示已有足够信息
- 要求直接给出建议

**处理**：
- 尊重用户选择
- 总结讨论成果
- 提供建议和资源

## 特殊情况处理

### 1. 用户方向不明

**问题**：用户没有明确研究兴趣

**策略**：
- 广泛探索
- 提供示例
- 识别兴趣点

**问题示例**：
```
"你最近在读什么论文？"
"你对哪个领域的问题最感兴趣？"
"你希望你的研究产生什么影响？"
```

### 2. 用户目标过大

**问题**：用户想研究的问题太大

**策略**：
- 识别核心问题
- 建议缩小范围
- 评估可行性

**问题示例**：
```
"这个问题涉及很多方面，你最想关注哪个？"
"你有什么资源来研究这么大的问题？"
"我们可以先从一个小问题开始吗？"
```

### 3. 用户目标过小

**问题**：用户想研究的问题太小

**策略**：
- 识别更广泛意义
- 连接更大问题
- 评估贡献度

**问题示例**：
```
"这个小问题有什么更广泛的意义？"
"这个发现能回答什么更大的问题？"
"这个研究对领域有什么贡献？"
```

### 4. 用户已有明确想法

**问题**：用户已经有明确研究问题

**策略**：
- 评估问题质量
- 检验假设和证据
- 提供改进建议

**问题示例**：
```
"你对这个问题有什么假设？"
"你有什么证据支持这个假设？"
"这个研究设计能回答你的问题吗？"
```

## 记录和追踪

### 1. 对话记录

**内容**：
- 用户原始想法
- 关键问题和回答
- 重要洞察
- 决策点

**格式**：
```
[日期] [轮次] [主题]
Q: 问题
A: 回答
Insight: 洞察
Decision: 决策
```

### 2. 进展追踪

**指标**：
- 问题明确度
- 假设检验度
- 证据充分度
- 用户满意度

**评估**：
- 每 10 轮评估一次
- 调整引导策略
- 记录进展

### 3. 输出生成

**内容**：
- 最终研究问题
- 讨论摘要
- 关键洞察
- 下一步建议

**格式**：
```
研究问题：[问题]
讨论摘要：[摘要]
关键洞察：[洞察列表]
下一步：[建议]
```
