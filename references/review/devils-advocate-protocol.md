# 魔鬼代言人协议

## 核心原则

### 1. 挑战假设

**目标**：识别和挑战隐含假设

**方法**：
- 质疑前提
- 提供反例
- 检验逻辑

### 2. 识别偏见

**目标**：检测确认偏见和其他认知偏见

**方法**：
- 寻找反面证据
- 考虑替代解释
- 评估选择性报告

### 3. 压力测试

**目标**：测试论点的稳健性

**方法**：
- 极端情况测试
- 边界条件分析
- 敏感性分析

## 挑战类型

### 1. 假设挑战

**目标**：识别和检验隐含假设

**问题示例**：
```
"你假设了什么？"
"这些假设基于什么证据？"
"如果这些假设不成立会怎样？"
"有没有其他可能的假设？"
```

**应用**：
- 研究设计假设
- 数据分析假设
- 结论解释假设

### 2. 证据挑战

**目标**：评估证据的充分性和可靠性

**问题示例**：
```
"你有什么证据支持这个结论？"
"这个证据可靠吗？"
"有没有反面证据？"
"证据是否充分？"
```

**应用**：
- 数据质量
- 样本代表性
- 测量有效性
- 统计方法

### 3. 逻辑挑战

**目标**：检验推理的逻辑一致性

**问题示例**：
```
"你的推理过程是什么？"
"这个推理有逻辑漏洞吗？"
"有没有其他可能的解释？"
"结论是否由证据支持？"
```

**应用**：
- 因果推理
- 归纳推理
- 演绎推理
- 类比推理

### 4. 替代解释

**目标**：考虑其他可能的解释

**问题示例**：
```
"有没有其他可能的解释？"
"这些解释有什么证据支持？"
"你如何排除这些解释？"
"哪个解释最合理？"
```

**应用**：
- 混杂因素
- 反向因果
- 选择偏见
- 测量误差

## 挑战流程

### Phase 1: 准备

**目标**：了解研究内容和论点

**活动**：
1. 阅读研究材料
2. 识别核心论点
3. 记录关键声明
4. 准备挑战问题

**输出**：
- 核心论点清单
- 关键声明列表
- 挑战问题准备

### Phase 2: 挑战

**目标**：系统性挑战研究论点

**活动**：
1. 假设挑战
2. 证据挑战
3. 逻辑挑战
4. 替代解释

**输出**：
- 挑战问题列表
- 识别的弱点
- 改进建议

### Phase 3: 评估

**目标**：评估挑战的结果

**活动**：
1. 评估论点稳健性
2. 识别需要改进的地方
3. 形成评估报告

**输出**：
- 稳健性评估
- 改进建议
- 最终报告

**挑战点**：

**挑战点**：
- 统计方法

- 效应量

**问题示例**：
```
"统计方法是否适当？"
"假设是否满足？"
"效应量是否有意义？"
```

**挑战点**：

- 外推性

```
"能否做因果推断？"
"结果能否外推？"
"临床意义是什么？"
"实践意义是什么？"
```

### 4. 报告完整性

**挑战点**：
- 选择性报告
- 负面结果
- 局限性
- 利益冲突

**问题示例**：
```
"是否所有结果都报告了？"
"负面结果是否报告？"
"局限性是否充分讨论？"
"利益冲突是否声明？"
```

## 挑战技巧

### 1. 提问技巧

**开放式问题**：
```
✅ "你如何解释这个现象？"
❌ "这个现象是因为X吗？"
```

**具体化问题**：
```
✅ "你有什么证据支持这个假设？"
❌ "这个假设合理吗？"
```

**挑战性问题**：
```
✅ "如果这个假设不成立，你的结论会怎样？"
❌ "你的假设是正确的吗？"
```

### 2. 倾听技巧

**积极倾听**：
- 复述对方观点
- 确认理解
- 表达认可

**回应方式**：
```
"我理解你的意思是..."
"你提到...，这很有趣"
"你的观点是..."
```

### 3. 平衡技巧

**挑战 vs. 支持**：
- 适度挑战
- 提供建设性反馈
- 承认优点

**深度 vs. 广度**：
- 深入关键问题
- 覆盖主要领域
- 平衡时间和精力

## 特殊情况处理

### 1. 对方防御性回应

**问题**：对方对挑战有防御性反应

**策略**：
- 保持冷静
- 重申建设性目的
- 提供具体证据
- 寻找共同点

**回应示例**：
```
"我理解你的立场。我的目的是帮助改进研究质量。让我们看看具体证据..."
```

### 2. 挑战过于严厉

**问题**：挑战过于严厉，影响关系

**策略**：
- 调整语气
- 承认优点
- 提供建设性建议
- 平衡挑战与支持

**回应示例**：
```
"你在这方面做得很好。同时，我们可以考虑..."
```

### 3. 挑战不足

**问题**：挑战不够深入，未能识别问题

**策略**：
- 增加挑战深度
- 寻找更多反例
- 考虑极端情况
- 压力测试论点

**回应示例**：
```
"让我们更深入地探讨这个问题。如果考虑极端情况..."
```
