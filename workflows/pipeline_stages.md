# Pipeline 阶段定义

## 核心原则

### 1. 阶段化

**要求**：
- 明确的阶段划分
- 清晰的阶段目标
- 标准化的阶段交接

### 2. 门禁化

**要求**：
- 关键阶段有门禁
- 门禁不可跳过
- 门禁有明确标准

### 3. 可追溯

**要求**：
- 记录阶段进展
- 记录决策过程
- 记录状态变化

---

## 10 阶段 Pipeline

### Stage 1: RESEARCH (研究)

**目标**：深度研究，建立知识基础

**活动**：
- 研究问题开发
- 文献搜索和筛选
- 来源验证和评估
- 综合分析

**输入**：
- 用户研究兴趣
- 初步想法
- 相关背景

**输出**：
- 研究问题
- 文献综述
- 研究空白
- 研究假设

**使用 Skill**：`deep_research`

**质量标准**：
- 研究问题满足 FINER
- 文献搜索系统
- 来源质量可靠
- 综合分析深入

### Stage 2: WRITE (写作)

**目标**：撰写论文初稿

**活动**：
- 论文结构设计
- 分章节撰写
- 引用整合
- 格式规范

**输入**：
- 研究成果
- 文献综述
- 实验数据
- 写作规范

**输出**：
- 论文初稿
- 参考文献
- 图表初稿
- 补充材料

**使用 Skill**：`academic_paper`

**质量标准**：
- 结构合理
- 内容完整
- 引用规范
- 格式正确

### Stage 2.5: INTEGRITY GATE (完整性门禁)

**目标**：预审完整性检查

**活动**：
- 7 模式 AI 研究失败清单检查
- 声明验证
- 数据完整性检查
- 引用完整性检查

**输入**：
- 论文初稿
- 实验数据
- 引用列表

**输出**：
- 完整性报告
- 问题清单
- 改进建议

**检查项**：
- [ ] 无数据伪造
- [ ] 无引用伪造
- [ ] 无抄袭
- [ ] 无利益冲突隐瞒
- [ ] 伦理合规
- [ ] 方法可重复
- [ ] 结果可验证

**决策**：
- PASS：进入下一阶段
- FAIL：返回 Stage 2 修改

### Stage 3: REVIEW (审稿)

**目标**：多视角同行评审

**活动**：
- 5 人审稿团组建
- 独立审稿
- 综合决策
- 修订建议

**输入**：
- 论文初稿
- 完整性报告

**输出**：
- 审稿报告
- 量化评分
- 修订建议
- 决策建议

**使用 Skill**：`academic_paper_reviewer`

**质量标准**：
- 5 人独立审稿
- 量化评分完整
- 修订建议具体
- 决策建议明确

**决策**：
- Accept：进入 Stage 5
- Minor Revision：进入 Stage 4
- Major Revision：进入 Stage 4
- Reject：返回 Stage 2 或终止

### Stage 4: REVISE (修订)

**目标**：根据审稿意见修订论文

**活动**：
- 分析审稿意见
- 制定修订计划
- 修订论文内容
- 准备回复信

**输入**：
- 审稿报告
- 修订建议

**输出**：
- 修订后论文
- 回复信
- 修订说明

**使用 Skill**：`academic_paper`

**质量标准**：
- 所有意见都有回复
- 修订有证据支持
- 回复信结构清晰
- 语言专业礼貌

### Stage 3': RE-REVIEW (复审)

**目标**：验证修订是否充分

**活动**：
- 审查修订内容
- 验证问题解决
- 评估修订质量
- 形成复审意见

**输入**：
- 修订后论文
- 回复信
- 原审稿报告

**输出**：
- 复审报告
- 修订验证
- 最终决策

**决策**：
- Accept：进入 Stage 5
- Minor Revision：进入 Stage 4'
- Reject：返回 Stage 2 或终止

### Stage 4': RE-REVISE (二次修订)

**目标**：根据复审意见进行二次修订

**活动**：
- 分析复审意见
- 修订论文内容
- 更新回复信
- 准备最终版本

**输入**：
- 复审报告
- 二次修订建议

**输出**：
- 最终论文
- 最终回复信
- 修订总结

### Stage 4.5: FINAL INTEGRITY (最终完整性)

**目标**：修订后完整性检查

**活动**：
- 全面完整性检查
- 零回归确认
- 最终质量评估
- 准备定稿

**输入**：
- 最终论文
- 修订历史

**输出**：
- 最终完整性报告
- 质量评估
- 定稿批准

**检查项**：
- [ ] 所有修订已实施
- [ ] 无新问题引入
- [ ] 质量无下降
- [ ] 完整性无问题

**决策**：
- PASS：进入 Stage 5
- FAIL：返回 Stage 4' 修改

### Stage 5: FINALIZE (定稿)

**目标**：最终格式化和输出

**活动**：
- 格式规范检查
- 引用格式统一
- 图表优化
- 输出最终版本

**输入**：
- 最终论文
- 格式规范

**输出**：
- 最终 PDF
- LaTeX 源文件
- 补充材料
- 投稿包

**质量标准**：
- 格式完全规范
- 引用完全正确
- 图表质量高
- 投稿包完整

### Stage 6: PROCESS SUMMARY (流程总结)

**目标**：生成流程总结报告

**活动**：
- 总结流程过程
- 评估协作质量
- 记录经验教训
- 生成最终报告

**输入**：
- 全流程记录
- 各阶段输出
- 质量评估

**输出**：
- 流程总结报告
- 协作质量评估
- 经验教训
- 改进建议

## 阶段交接

### 交接规范

**交接内容**：
- 阶段输出
- 质量评估
- 问题清单
- 改进建议

**交接格式**：
```
阶段：[名称]
状态：[完成/失败]
输出：[文件列表]
质量：[评估]
问题：[问题清单]
建议：[改进建议]
```

### 交接检查

**检查项**：
- [ ] 输出完整
- [ ] 质量达标
- [ ] 问题已解决
- [ ] 建议已记录

## 状态管理

### 状态类型

**阶段状态**：
- Not Started：未开始
- In Progress：进行中
- Completed：已完成
- Failed：已失败
- Skipped：已跳过

**门禁状态**：
- Pending：待检查
- Passed：已通过
- Failed：未通过

**流程状态**：
- Active：活跃
- Paused：暂停
- Completed：已完成
- Terminated：已终止

### 状态转换

**正常流程**：
```
Stage 1 → Stage 2 → Stage 2.5 → Stage 3 → Stage 4 → Stage 3' → Stage 4' → Stage 4.5 → Stage 5 → Stage 6
```

**分支流程**：
```
Stage 3 (Accept) → Stage 5
Stage 3 (Reject) → Stage 2 或终止
Stage 3' (Accept) → Stage 5
Stage 3' (Reject) → Stage 2 或终止
```

**失败流程**：
```
Stage 2.5 (FAIL) → Stage 2
Stage 4.5 (FAIL) → Stage 4'
```

## 质量门禁

### Stage 2.5 门禁

**检查内容**：
- 数据完整性
- 引用完整性
- 伦理合规性
- 方法可重复性

**通过标准**：
- 所有检查项通过
- 无严重问题
- 问题可修复

**失败处理**：
- 返回 Stage 2
- 修复问题
- 重新检查

### Stage 3 门禁

**检查内容**：
- 审稿完整性
- 评分合理性
- 建议可行性
- 决策明确性

**通过标准**：
- 5 人审稿完成
- 评分一致
- 建议具体
- 决策明确

**失败处理**：
- 重新审稿
- 补充审稿人
- 重新评估

### Stage 4.5 门禁

**检查内容**：
- 修订完整性
- 零回归确认
- 质量无下降
- 完整性无问题

**通过标准**：
- 所有修订已实施
- 无新问题
- 质量无下降
- 完整性无问题

**失败处理**：
- 返回 Stage 4'
- 修复问题
- 重新检查
